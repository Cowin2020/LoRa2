Data type and constants
-----------------------
	Type of packet (1 byte)
		0: TIME
		1: ACK
		2: SEND
	Device ID (1 byte)
		Each sender device has a unique ID.
	Nonce (96 bits, 12 bytes)
		Initialization vector randomize for each packet.
	Serial code (4 bytes, natural number)
		Monotonic increasing
		Allow roll-back only if MSB is 1.
	Time (20 bytes, ASCII string)
		ISO8601 format with "Z" timezone

Protocol of time synchronization
--------------------------------
	1. Server: type 0 TIME, nonce, encrypted current time, authentication tag.

Protocol of uploading data
--------------------------
	1. Client: type SEND, 1 byte client ID, nonce, encrypted serial code and value, authentication tag.
	2. Server: type ACK, 1 byte client ID, nonce, encrypted serial code, authentication tag.
	3. If ACK is not receviced from server and number of tries is not over limit, loop back to step 1.
