Data type and constants
-----------------------
	Type of packet (1 byte)
		0: TIME
		1: ACK
		2: SEND
	Device ID (1 byte)
		Each sender device has a unique ID.
	Nonce (96 bits, 12 bytes)
		Initialization vector randomize for each packet.
	Serial code (4 bytes, natural number)
		Monotonic increasing
		Allow roll-back only if MSB is 1.
	Values (0-20 bytes, binary format)
		4 bytes float temperature from Dallas thermometer if ENABLE_DALLAS
		4 bytes float temperature from BME280 if ENABLE_BME
		4 bytes float pressure from BME280 if ENABLE_BME
		4 bytes float humidity from BME280 if ENABLE_BME
		4 bytes float ultraviolet LTR390 sensor if ENABLE_LTR
	Time (7 bytes, binary format)
		2 bytes year
		1 byte month
		1 byte day
		1 byte hour
		1 byte minute
		1 byte second

Protocol of time synchronization
--------------------------------
	1. Server: type 0 TIME, nonce, encrypted current time, authentication tag.

Protocol of uploading data
--------------------------
	1. Client: type SEND, 1 byte client ID, nonce, encrypted serial code, values, time, and authentication tag.
	2. Server: type ACK, 1 byte client ID, nonce, encrypted serial code, and authentication tag.
	3. If ACK is not receviced from server and number of tries is not over limit, loop back to step 1.
